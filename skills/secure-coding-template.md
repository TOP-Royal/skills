# 安全编码模板（secure-coding）

> 目标：把高频安全问题在编码阶段就消灭。

## 固定输出物（AI 必须产出）

- 威胁点清单（入口、资产、攻击面）
- 防护点清单（输入校验/鉴权授权/脱敏/限流/审计）
- 敏感字段清单（存储/传输/日志策略）
- 依赖与扫描动作（SCA/代码扫描/镜像扫描）

## 必须/禁止规则

- 必须：所有外部输入做白名单校验（类型/范围/长度）
- 必须：关键动作做服务端权限校验（不信任前端）
- 禁止：secret 写入代码/提交到仓库/打印到日志
- 禁止：直接拼接 SQL/命令/路径

---

## 1. 输入校验

- 白名单校验（类型、范围、长度、枚举）
- 防注入：SQL 参数化、命令执行禁用拼接
- 文件/路径：防路径遍历（../）

---

## 2. 授权与鉴权

- 服务端鉴权（不信任前端）
- RBAC/ABAC 权限模型
- 多租户隔离（tenant_id）

---

## 3. 数据保护

- 密码：强散列（bcrypt/argon2）
- 敏感数据：加密存储/传输
- 日志脱敏：手机号/证件号/token
- Secret 管理：环境变量/Secret Manager，不入库不入 Git

---

## 4. Web 常见漏洞

- XSS：输出编码、CSP
- CSRF：SameSite、CSRF token
- SSRF：禁止内网探测、域名白名单
- 文件上传：MIME 校验、大小限制、病毒扫描

---

## 5. 依赖与供应链

- SCA：Dependabot/Snyk
- 镜像扫描：Trivy
- 代码扫描：Semgrep

---

## 6. 验收清单（DoD）

- [ ] 输入校验覆盖所有外部入口
- [ ] 权限校验在关键动作处都有
- [ ] 日志无敏感信息
- [ ] 依赖扫描无高危（或有豁免理由）
