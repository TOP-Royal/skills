# 数据库设计模板（db-design）

> 目标：表结构/索引/事务/迁移一次设计正确，减少线上性能和一致性问题。

## 固定输出物（AI 必须产出）

- 表结构定义（字段/类型/默认值/NULL/约束/说明）
- 索引清单（每个索引服务于哪个查询）
- 关键查询的 EXPLAIN 结论（期望走的索引/是否可能回表/是否全表扫）
- 并发写入策略（乐观锁/悲观锁/唯一约束幂等）
- Migration 方案（up/down）+ 回滚演练步骤

## 必须/禁止规则

- 必须：所有关键查询都有索引支撑并验证执行计划
- 必须：迁移提供回滚路径（down 或等价回退方案）
- 必须：并发写入有明确冲突处理（避免重复数据/丢失更新）
- 禁止：在大表上新增高成本索引而无评估
- 禁止：长事务包住网络调用/复杂计算
