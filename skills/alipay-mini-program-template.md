# 支付宝小程序开发模板（alipay-mini-program）

> 目标：支付宝小程序从开发到蚂蚁开放平台一次成型，覆盖商业服务、支付、信用、会员等阿里生态核心能力。

---

## 固定输出物（AI 必须产出）

- 产品定位与支付宝生态契合点（生活服务/商业/政务/金融/会员）
- 页面与信息架构（首页/服务/我的，符合支付宝用户心智）
- 核心能力接入方案（登录/支付/信用/会员/卡券/营销）
- 数据与埋点设计（MyBank 数据规范/支付宝统计/自定义事件）
- 性能与体验基线（启动/包体/渲染/内存）
- 审核与上架材料（营业执照/行业资质/隐私政策/功能说明）

## 必须/禁止规则

- 必须：支付宝小程序必须以"服务交易"或"生活便利"为核心场景，纯内容展示型易被拒
- 必须：所有支付走支付宝收银台（my.tradePay），禁止引导用户外部支付
- 必须：用户授权（手机号/地址/会员信息）有明确用途说明，拒绝后功能可降级
- 必须：性能基线：首屏 < 1.5s（低端机）、包体 < 2MB（主包）、JS 执行 < 2s
- 禁止：诱导用户关注/收藏/跳转其他小程序（支付宝审核严格限制）
- 禁止：收集与业务无关的用户信息（过度授权会被拒绝上架）
- 禁止：使用非支付宝官方支付渠道或引导至 H5 外部支付
- 禁止：页面空白/加载超时无反馈（必须有骨架屏或 loading 态）

## 1. 平台特性与生态定位

- 核心场景：生活服务、商业交易、政务办事、金融理财、会员运营
- 用户心智：打开支付宝 → 找服务/付钱/办事（工具属性强，非社交内容）
- 流量入口：
  - 首页搜索 / 应用中心 / 生活号
  - 扫码 / 分享 / 消息推送
  - 卡包 / 会员 / 优惠券
- 与微信差异：
  - 更强交易属性，弱社交属性
  - 信用能力（芝麻信用）是独特优势
  - 会员通（与天猫/淘宝会员体系打通）

---

## 2. 技术架构

- 框架：基于支付宝小程序原生框架（类似微信小程序，API 有差异）
- 语言：JavaScript / TypeScript（官方支持 TS）
- 开发工具：支付宝小程序开发者工具
- 组件：my.xxx API（与 wx.xxx 类似但命名不同）
- 状态管理：官方无强制方案，可自研或使用第三方（如 mobx-miniprogram）
- 多端复用：UniApp / Taro 支持编译到支付宝（需注意平台差异）

---

## 3. 核心能力接入

### 3.1 登录与授权

- my.getAuthCode（获取授权码）
- my.getOpenUserInfo（获取用户基础信息）
- my.getPhoneNumber（获取手机号，需用户点击按钮触发）
- 授权原则：首次使用功能时申请，拒绝后提供降级方案

### 3.2 支付

- my.tradePay（唤起支付宝收银台）
- 接入流程：
  - 服务端创建订单 → 获取 tradeNO → 客户端调用 my.tradePay
  - 服务端主动查询订单状态（防掉单）
- 退款：服务端调用支付宝退款接口
- 分账：如涉及平台抽成，使用支付宝分账功能

### 3.3 信用能力（独特优势）

- 芝麻信用：信用免押、信用代扣、信用评估
- 花呗分期：接入花呗分期支付
- 先用后付：信用购/先用后付场景

### 3.4 会员与营销

- 支付宝会员：积分/等级/权益打通
- 卡券：优惠券/代金券/兑换券（接入支付宝卡包）
- 营销：满减/折扣/拼团/直播（支付宝生活号+小程序联动）

### 3.5 开放能力

- 扫一扫：my.scanCode
- 位置服务：my.getLocation / my.openLocation
- 图片/文件：my.chooseImage / my.uploadFile
- 消息：my.subscribeMessage（订阅消息，用户主动触发）

---

## 4. 页面与组件规范

- 页面结构：.js / .json / .axml（模板）/ .acss（样式）
- 常用组件：view / text / image / button / input / scroll-view / swiper
- 导航：my.navigateTo / my.redirectTo / my.reLaunch / my.navigateBack
- 与微信小程序差异：
  - axml 与 wxml 语法基本一致
  - acss 与 wxss 基本一致
  - API 前缀 my 替代 wx
  - 部分组件属性名不同（需查官方文档）

## 5. 审核与上架

### 5.1 资质要求

- 企业营业执照（个体户有限制）
- 法人身份证
- 行业资质（如食品经营许可证、ICP 备案、医疗资质等，按行业）
- 支付宝企业认证

### 5.2 审核要点

- 功能完整性：页面无空链、无占位符、无测试数据
- 内容合规：无虚假宣传、无敏感内容、无诱导分享
- 支付合规：虚拟商品/服务需走支付宝，禁止外部支付引导
- 隐私合规：隐私政策完整，权限申请有正当理由

### 5.3 上架流程

1. 开发完成 → 自测（开发者工具真机预览）
2. 提交审核（填写功能说明、上传截图、勾选能力）
3. 审核中（1-5 个工作日）
4. 审核通过 → 发布上线
5. 版本迭代：灰度发布 → 全量发布

---

## 6. 性能与体验

- 启动优化：
  - 主包 ≤ 2MB，分包 ≤ 4MB
  - 首屏 < 1.5s（低端安卓机）
  - 预加载 / 骨架屏 / 数据缓存
- 渲染优化：
  - 列表虚拟滚动（长列表）
  - 图片懒加载 / 压缩 / CDN
  - 减少 setData 频次和数据量
- 内存优化：
  - 页面卸载时清理定时器/监听器
  - 大图及时释放
