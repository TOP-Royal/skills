# 抖音小程序开发模板（douyin-mini-program）

> 目标：抖音小程序从开发到字节开放平台一次成型，覆盖内容分发、电商、直播、社交裂变等字节生态核心能力。

---

## 固定输出物（AI 必须产出）

- 产品定位与抖音生态契合点（内容/电商/工具/游戏/本地生活）
- 流量获取策略（短视频挂载/直播挂载/搜索/分享/广告投放）
- 页面与信息架构（首页/详情/交易/个人中心，符合抖音内容消费习惯）
- 核心能力接入方案（登录/支付/分享/拍摄/直播/广告投放）
- 内容合规与审核策略（UGC/PGC 内容分级、版权、未成年人保护）
- 性能与体验基线（启动/包体/视频播放/内存）
- 上架与运营材料（营业执照/行业资质/隐私政策/内容安全承诺书）

## 必须/禁止规则

- 必须：抖音小程序必须与"内容"或"交易"强相关，纯工具型需证明有内容传播价值
- 必须：视频/图片内容有版权证明或原创声明，禁止搬运/侵权
- 必须：涉及 UGC 的必须有审核机制（先发后审或先审后发）和内容举报入口
- 必须：未成年人相关（游戏/直播/社交）需有防沉迷/家长监护/实名认证
- 必须：性能基线：首屏 < 1.5s、视频首帧 < 500ms、包体 < 2MB（主包）
- 禁止：诱导分享/裂变（如"分享给好友才能继续"）违反抖音平台规则
- 禁止：站外引流（引导用户加微信/下载 App/跳转 H5 外部交易）
- 禁止：虚假宣传/夸大效果/伪造用户评价（抖音审核对电商类尤其严格）

## 1. 平台特性与生态定位

- 核心场景：内容消费、短视频电商、直播带货、本地生活、小游戏、社交娱乐
- 用户心智：刷视频 → 被种草 → 直接购买/预约/互动（内容即交易）
- 流量入口：
  - 短视频挂载（视频左下角链接）
  - 直播挂载（直播间小雪花/购物车）
  - 搜索 / 抖音主页 / 消息分享
  - 广告投放（巨量引擎）
- 独特优势：
  - 内容+交易闭环（视频种草 → 小程序成交）
  - 算法推荐（优质内容获得更多流量）
  - 直播能力（与直播深度打通）

---

## 2. 技术架构

- 框架：字节小程序原生框架（基于字节小程序 DSL）
- 语言：JavaScript / TypeScript
- 开发工具：字节跳动开发者工具
- 组件：tt.xxx API（与 wx.xxx 类似）
- 多端复用：UniApp / Taro 支持编译到抖音（注意条件编译处理平台差异）

---

## 3. 核心能力接入

### 3.1 登录与授权

- tt.login（获取 code）→ 服务端换 session
- tt.getUserInfo（获取用户昵称/头像，需用户点击按钮）
- tt.getPhoneNumber（获取手机号，需用户主动授权）

### 3.2 支付

- tt.pay（抖音支付）
- 接入流程：服务端创建订单 → 获取预支付信息 → 客户端调 tt.pay
- 退款：服务端调用抖音退款接口
- 注意：虚拟商品/服务需走抖音支付，禁止引导外部支付

### 3.3 内容能力（核心差异）

- 短视频挂载：
  - 发布视频时挂载小程序页面
  - 视频内容与小程序内容强相关（挂羊头卖狗肉会被限流）
- 拍摄器：tt.openCamera / tt.chooseMedia
- 直播挂载：直播间组件/小雪花/购物车与小程序打通
- 视频播放：Video 组件，支持全屏/小窗/弹幕

### 3.4 社交与分享

- tt.shareAppMessage（分享给好友/群，抖音内）
- 口令分享：生成口令 → 用户复制 → 打开抖音跳转小程序
- 二维码：小程序码/普通链接二维码
- 注意：禁止诱导分享（如"分享得红包"可能违规）

### 3.5 广告投放

- 巨量引擎：短视频广告/直播间广告/搜索广告
- 转化追踪：小程序内事件回传（注册/下单/支付）
- 人群包：DMP 数据对接，精准投放

---

## 4. 内容合规与审核

### 4.1 UGC/PGC 内容审核

- 先发后审 vs 先审后发（根据业务风险等级选择）
- 内容分级：公开/好友可见/私密
- 敏感词过滤：文字/图片/视频 AI 审核 + 人工复审
- 举报机制：每条内容需有举报入口

### 4.2 电商合规

- 商品信息真实（禁止虚假宣传）
- 价格合规（不得虚构原价）
- 七天无理由退货（符合消保法）
- 资质：食品/化妆品/医疗器械等特殊类目需许可证

### 4.3 未成年人保护

- 实名认证：涉及支付/直播/社交需实名
- 防沉迷：游戏/直播时长限制
- 家长监护：未成年人消费限制与退款机制

---

## 5. 页面与组件规范

- 页面结构：.js / .json / .ttml（模板）/ .ttss（样式）
- 常用组件：view / text / image / button / input / scroll-view / swiper / video
- 导航：tt.navigateTo / tt.redirectTo / tt.reLaunch / tt.navigateBack
- 与微信小程序差异：
  - ttml 与 wxml 语法基本一致
  - ttss 与 wxss 基本一致
  - API 前缀 tt 替代 wx
  - Video 组件能力更强（支持全屏/弹幕/小窗）

---

## 6. 性能与体验

- 启动优化：
  - 主包 ≤ 2MB，分包 ≤ 4MB
  - 首屏 < 1.5s
  - 骨架屏 / 预加载 / 数据缓存
- 视频优化：
  - 首帧 < 500ms
  - 预加载策略（WiFi 下自动预加载）
  - 清晰度自适应（根据网络切换）
- 内存优化：
  - 视频及时销毁（页面不可见时暂停/释放）
  - 大图/长列表虚拟滚动

---

## 7. 审核与上架

### 7.1 资质要求

- 企业营业执照（个体工商户部分类目受限）
- 法人身份证
- ICP 备案（如涉及网络内容服务）
- 行业资质（按类目：食品/医疗/教育/金融等）
- 内容安全承诺书（UGC 类必须）

### 7.2 审核要点

- 内容合规：无色情/暴力/政治敏感/虚假宣传
- 功能完整：无测试数据/空页面/占位符
- 支付合规：走抖音支付，无站外引流
- 未成年人保护：游戏/直播/社交需实名+防沉迷

### 7.3 上架流程

1. 开发完成 → 自测（开发者工具真机预览）
2. 提交审核（上传代码/填写功能说明/资质材料）
3. 审核中（1-7 个工作日，UGC/电商类可能更长）
4. 审核通过 → 发布上线
5. 版本迭代：灰度 → 全量
