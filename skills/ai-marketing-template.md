# AI 营销能力模板（ai-marketing）

> 目标：利用 AI 提升营销效率与转化率，覆盖内容生成、用户洞察、个性化触达、投放优化与数据闭环。

---

## 固定输出物（AI 必须产出）

- 营销策略框架（目标受众 / 价值主张 / 渠道组合 / 内容矩阵 / 转化漏斗）
- AI 内容生成规范（Prompt 模板 / 品牌调性 / 多平台适配 / 合规审查清单）
- 用户分层与个性化策略（RFM / 行为标签 / 预测模型 / 触达时机 / 渠道偏好）
- 投放与 A/B 测试方案（实验假设 / 分流策略 / 观察指标 / 成功标准 / 样本量计算）
- 营销数据指标体系（北极星指标 / 过程指标 / 归因模型 / 数据看板清单）
- AI 工具选型与集成方案（内容生成 / 用户分析 / 智能客服 / 自动化流程）

## 必须/禁止规则

- 必须：所有 AI 生成内容经过品牌调性审核 + 事实核查 + 合规审查（禁用词 / 版权 / 隐私）
- 必须：用户数据用于 AI 模型前脱敏/匿名化，符合 GDPR/个人信息保护法
- 必须：A/B 测试有明确假设和样本量计算，禁止"跑几天看看"
- 必须：营销自动化有退订/关闭入口，禁止骚扰式推送
- 禁止：AI 生成内容直接发布不做人工审核（尤其医疗/金融/法律等强监管行业）
- 禁止：用用户隐私数据训练公开模型或泄露给第三方
- 禁止：虚假宣传/夸大效果/伪造用户评价（AI 生成评论/案例需标注）
- 禁止：无归因模型就判断"某个渠道无效"（需多触点归因）

## 2. AI 内容生成

### 2.1 内容类型与 Prompt 规范

- 品牌文案：
  - 调性定义：专业/活泼/高端/亲民（给出具体词汇和禁用词）
  - Prompt 模板："以 [调性] 风格，面向 [受众]，突出 [卖点]，生成 [格式]…"
- 多平台适配：
  - 小红书：emoji + 痛点 + 解决方案 + 互动引导
  - 抖音：黄金3秒 + 故事线 + 行动号召
  - 公众号：标题党检测 + 结构化排版 + 阅读完成率优化
  - B站：知识密度 + 弹幕互动点 + 系列化
- SEO 内容：
  - 关键词研究（AI 辅助：长尾词/意图分析/竞品覆盖）
  - 内容结构（Featured Snippet 优化/目录/FAQ）

### 2.2 AI 辅助创意工具链

- 文案生成：ChatGPT/Claude/通义千问/文心一言
- 图片生成：Midjourney/Stable Diffusion/DALL-E/即梦
- 视频生成：剪映/Runway/Pika/数字人
- 多语言本地化：DeepL/ChatGPT + 本地文化审核

### 2.3 内容审核与合规

- 品牌一致性：AI 生成 → 品牌调性检查清单 → 人工终审
- 事实核查：数据/引用/案例来源可追溯
- 合规审查：
  - 禁用词库（广告法极限词、医疗/金融敏感词）
  - 版权检查（图片/音乐/字体授权）
  - 隐私合规（用户案例需授权，数据脱敏）
- AI 标识：按平台要求标注"AI生成"或"广告"（如需要）

## 3. 用户洞察与分层

### 3.1 数据收集与整合

- 第一方数据：
  - 行为数据：浏览/搜索/点击/加购/购买/复购
  - 交易数据：客单价/频次/品类偏好/支付渠道
  - 互动数据：邮件打开/推送点击/客服对话/评价
- 第二方数据：合作平台脱敏数据
- 第三方数据：合规获取的人口/兴趣/消费力标签

### 3.2 AI 用户分群模型

- RFM 模型：Recency(最近购买) / Frequency(频次) / Monetary(金额)
- 行为聚类：
  - 高价值活跃用户
  - 潜在升级用户
  - 流失预警用户
  - 新客培育用户
- 预测模型：
  - 购买概率预测（Next Best Action）
  - 流失概率预测（干预时机）
  - LTV 预测（投放预算分配）

### 3.3 用户标签体系

| 标签类型 | 示例 | 来源 |
|---------|------|------|
| 基础属性 | 年龄/性别/城市/设备 | 注册/设备信息 |
| 行为标签 | 夜间浏览/比价型/冲动型 | 行为序列分析 |
| 兴趣标签 | 科技/母婴/美妆/游戏 | 内容互动/搜索词 |
| 价值标签 | 高净值/价格敏感/品质导向 | 交易数据 |
| 生命周期 | 新客/成长/成熟/衰退/流失 | RFM + 行为 |

## 4. 个性化触达与营销自动化

### 4.1 触达策略

- 时机优化：
  - 个体最佳发送时间（AI 预测打开率最高的时段）
  - 事件触发：浏览未购买 / 加购未支付 / 生日 / 沉默 30 天
- 渠道选择：
  - 消息类型 → 最佳渠道（邮件/Push/短信/微信/电话/站内信）
  - 用户渠道偏好学习（哪个渠道对该用户 CTR 最高）
- 内容个性化：
  - 动态插入：姓名/最近浏览商品/专属优惠/地理位置
  - 推荐商品：协同过滤 / 内容相似 / 热门 / 互补

### 4.2 营销自动化流程（Journey）

```
触发事件 → 条件判断 → 动作执行 → 等待/分支 → 下一动作
```

- 新客旅程：注册 → 欢迎系列 → 首单引导 → 首购关怀 → 复购激励
- 沉默唤醒：沉默 7 天 → 内容触达 → 沉默 30 天 → 专属优惠 → 沉默 60 天 → 问卷/电话
- 购物车挽回：加购 1h → 提醒 → 24h → 小额优惠 → 72h → 限时折扣 → 7 天 → 相似推荐

### 4.3 智能客服与销售辅助

- AI 客服：常见问题自动回复 / 意图识别 / 情绪分析 / 人工转接
- 销售辅助：客户画像实时推送 / 话术推荐 / 成交概率预测
- 合规：敏感问题转人工 / 金融/医疗建议免责声明

## 5. 投放优化与 A/B 测试

### 5.1 投放策略

- 渠道评估矩阵：
  | 渠道 | CAC | LTV | 转化周期 | 规模潜力 | 当前优先级 |
  |------|-----|-----|---------|---------|-----------|
- 预算分配：
  - 测试期：小规模多组创意测试
  - 放量期：赢家素材扩量
  - 稳定期：ROI 监控与衰减预警
- AI 辅助优化：
  - 自动出价（oCPM / tCPA）
  - 创意组合优化（标题 × 图片 × 受众自动测试）
  - 落地页热力图 + 转化漏斗分析

### 5.2 A/B 测试规范

- 实验设计：
  - 假设："如果 [改动]，那么 [指标] 会 [变化]，因为 [原因]"
  - 变量：单变量原则（一次只改一个元素）
  - 对照组：当前版本 / 默认策略
- 分流：
  - 用户级分流（user_id hash，保证同一用户始终同组）
  - 流量占比：通常 50/50 或 80/20（低风险）
  - 样本量计算：基于基线转化率、最小可检测效果(MDE)、显著性水平(α=0.05)、统计功效(1-β=0.8)
- 观察指标：
  - 核心指标（OEC，Overall Evaluation Criterion）：1 个
  - 护栏指标（Guardrail Metrics）：不能损害的指标（如退款率、投诉率）
  - 辅助指标：过程指标，帮助理解原因
- 实验周期：至少 1 个完整业务周期（如 7 天覆盖周末效应）
- 结果判定：
  - 统计显著 + 业务显著（实际提升幅度有价值）
  - 负面结果同样记录（避免重复踩坑）

## 6. 数据分析与归因

### 6.1 核心指标体系

- 北极星指标（1 个）：
- 过程指标（漏斗各环节）：
- 健康指标（长期）：NPS / 客诉率 / 品牌搜索量

### 6.2 归因模型

- 首次点击归因（拉新权重）
- 末次点击归因（收割权重）
- 线性归因（均匀分配）
- 位置归因（首次和末次各占 40%，中间均分 20%）
- 数据驱动归因（Shapley Value / Markov Chain）
- 禁止：单看末次归因就砍掉"助攻"渠道

### 6.3 营销数据看板

- 实时看板：投放消耗/CTR/CVR/ROI（小时级）
- 日报：渠道效果/内容表现/用户分层变动
- 周报：实验结果/趋势分析/下周计划
- 月度复盘：ROI / CAC / LTV / 品牌指标 / 策略调整

## 8. 验收清单（DoD）

- [ ] 营销策略框架完整（受众/价值主张/渠道/漏斗）
- [ ] AI 内容生成有品牌调性审核 + 事实核查 + 合规审查流程
- [ ] 用户分层模型可解释（RFM/标签定义清晰，非黑盒）
- [ ] 个性化触达有退订/关闭入口，频次受控
- [ ] A/B 测试有明确假设、样本量计算、护栏指标
- [ ] 数据归因模型明确，不依赖单点归因做渠道决策
- [ ] 用户数据使用符合 GDPR/个人信息保护法（脱敏/授权/最小必要）
- [ ] 营销自动化 Journey 已测试全分支（正常/异常/边界）
