# 技能可执行化标准（Skill Execution Standard）

> 所有 `skills/` 下的详细技能文档（01-19 及后续新增）必须遵循本标准，才能从“知识介绍”升级为“可执行交付”。

---

## 强制头部（每个 .md 文件开头必须包含）

每个技能文档必须在文件最开头（标题之后）插入以下两段：

```markdown
## 固定输出物（AI 必须产出）

- 输出物 1：
- 输出物 2：
- 输出物 3：

## 必须/禁止规则

- 必须：规则 1
- 必须：规则 2
- 禁止：规则 3
```

> **门禁**：未产出“固定输出物”之前，禁止进入代码实现阶段。

---

## 强制尾部（每个 .md 文件末尾必须包含）

```markdown
---

## 验收清单（DoD）

- [ ] 固定输出物 1 已产出并可验证
- [ ] 固定输出物 2 已产出并可验证
- [ ] 必须规则已检查通过
- [ ] 常见失败场景已列出并覆盖
- [ ] 最小可运行示例/测试已提供
```

---

## 输出物类型参考

根据技能类型选择对应的输出物：

| 技能类型 | 典型输出物 |
|---------|-----------|
| 编程语言 | 环境配置清单 + 最小可运行示例 + 核心概念速查表 |
| 前端框架 | 组件清单 + 路由/状态设计 + 样式规范 + 性能指标 |
| 后端框架 | API 清单 + 数据库迁移脚本 + 依赖版本锁 + 部署配置 |
| AI/LLM | Prompt 模板 + RAG 管道配置 + 评估用例 + 模型版本 |
| 云原生 | Terraform/Helm 片段 + 监控告警规则 + CI/CD 流水线片段 |
| 数据工程 | SQL/ETL 脚本 + 数据质量检查 + Schema 变更脚本 |
| 安全 | 威胁清单 + 防护清单 + 扫描结果 + 合规检查表 |

---

## 必须/禁止规则通用基线

所有技能文档至少包含以下 3 条“必须”和 2 条“禁止”：

- **必须**：关键路径有测试或验证手段（不能纯理论）
- **必须**：产出物包含版本/环境信息（防止“在我机器上能跑”）
- **必须**：错误场景有明确处理策略（不要只写 happy path）
- **禁止**：直接复制粘贴无验证的代码片段
- **禁止**：遗漏安全/权限/性能约束（任何技能都必须提）

## 与模板体系的调用关系

当使用某个详细技能时，必须联动调用以下模板（如适用）：

- 涉及 API → 联动 `api-design-template.md`
- 涉及数据库 → 联动 `db-design-template.md`
- 涉及外部输入/权限 → 联动 `secure-coding-template.md`
- 涉及上线/关键链路 → 联动 `observability-template.md`
- 任何功能 → 联动 `testing-strategy-template.md`
- 提交前 → 联动 `definition-of-done-template.md`

> 调用方式：在技能文档的“相关模板”章节中明确写出联动关系。
