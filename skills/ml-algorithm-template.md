# 机器学习与算法工程能力模板（ml-algorithm）

> 目标：推荐/搜索/预测模型从需求定义到线上服务一次成型，覆盖特征工程、模型训练、评估、部署、监控与迭代。

---

## 固定输出物（AI 必须产出）

- 业务问题定义与算法选型（目标函数/约束/评估指标/基线方案）
- 数据与特征方案（数据源/特征库/特征工程/在线/离线一致性）
- 模型训练与实验管理（实验追踪/超参搜索/版本管理/可复现）
- 评估方案（离线指标/在线 A-B 测试/业务指标/公平性/鲁棒性）
- 模型部署与服务化（推理延迟/吞吐/批处理/实时/模型版本管理）
- 监控与漂移检测（数据漂移/概念漂移/性能衰减/自动回滚）
- 迭代计划（反馈闭环/重训练触发条件/特征/模型迭代节奏）

## 必须/禁止规则

- 必须：任何模型上线前必须有明确的业务目标、评估指标、基线对比（新模型必须显著优于基线或业务规则）
- 必须：特征在线/离线一致性校验（训练-服务偏差是线上效果差的头号原因）
- 必须：模型版本管理（代码/数据/特征/参数/环境全部版本化，可复现）
- 必须：A-B 测试有样本量计算、实验时长、成功标准、护栏指标
- 必须：推理服务有超时/降级/熔断策略，模型异常时自动回退到基线
- 禁止：无评估指标就训练模型（"感觉效果好"不可接受）
- 禁止：数据泄露（测试集信息流入训练/特征工程）
- 禁止：直接用全量数据训练后上线，无 A-B 测试或灰度
- 禁止：模型上线后无监控（数据漂移/性能衰减导致效果劣化无人知晓）

## 1. 业务问题定义与算法选型

### 1.1 问题分类

| 类型 | 场景 | 典型算法 | 评估指标 |
|------|------|---------|---------|
| 推荐 | 商品/内容/视频推荐 | 协同过滤 / 深度学习（DIN/DeepFM） | CTR/CVR/GMV/留存/多样性 |
| 搜索 | 文本/电商/企业内部搜索 | BM25 / Learning to Rank（LambdaMART/DL） | NDCG/MRR/CTR/转化率 |
| 预测 | 销量/流量/风险/流失 | 时序（ARIMA/Prophet/LSTM）/ 树模型（XGB/LightGBM） | MAE/RMSE/AUC/F1 |
| 分类 | 风控/审核/标签 | 逻辑回归 / 树模型 / BERT | AUC/F1/Precision/Recall |
| 聚类 | 用户分群/异常检测 | K-Means / DBSCAN / 孤立森林 | Silhouette / 业务解释性 |

### 1.2 基线方案

- 规则基线：按热度/时间/价格排序（推荐/搜索）
- 统计基线：历史均值/滑动平均（预测）
- 简单模型：逻辑回归 / 线性回归
- 新模型必须：**统计显著 + 业务显著** 优于基线

## 2. 数据与特征工程

### 2.1 数据源

- 用户特征：画像/行为序列/偏好/社交关系
- 物品特征：内容/类目/价格/库存/质量分
- 上下文特征：时间/地点/设备/场景/天气
- 交互特征：点击/购买/收藏/分享/停留时长

### 2.2 特征工程

- 数值：归一化 / 标准化 / 分箱 / log 变换
- 类别：One-Hot / Embedding / Target Encoding（防泄露）
- 时序：滑动窗口 / 趋势 / 季节性 / 滞后特征
- 文本：TF-IDF / Word2Vec / BERT embedding
- 交叉特征：FM / DeepFM 自动学习 / 手工笛卡尔积

### 2.3 特征一致性（关键）

- 在线/离线特征工程代码必须同构（同一套特征生成逻辑）
- 特征存储：
  - 离线：Hive/Parquet + 特征库（Feathr/Feature Store）
  - 在线：Redis / Feature Store（Tecton/Feast/ByteDance Feature Store）
- 校验：每日对比在线特征值 vs 离线特征值（误差 < 1%）

## 3. 模型训练与实验管理

### 3.1 实验追踪

- 工具：MLflow / Weights & Biases / Neptune / TensorBoard
- 记录：代码版本/数据版本/特征集/超参数/指标/模型 artifact

### 3.2 超参搜索

- 网格搜索 / 随机搜索 / Bayesian Optimization / Optuna
- 交叉验证：时间序列用前向验证（避免数据泄露）

### 3.3 可复现性

- 固定随机种子
- 环境 Docker 化（requirements.txt / poetry / conda env）
- 数据版本（DVC / LakeFS / 数据快照）

## 4. 模型评估

### 4.1 离线评估

| 任务 | 指标 | 说明 |
|------|------|------|
| 分类 | AUC / F1 / Precision / Recall / PR Curve | 关注业务关注的类别 |
| 回归 | MAE / RMSE / MAPE / R² | 关注极端误差 |
| 排序 | NDCG / MAP / MRR | 关注头部结果质量 |
| 推荐 | HR / Recall / Precision / Coverage / Diversity | 避免过度集中 |

### 4.2 在线 A-B 测试

- 假设："新模型在指标 X 上提升 ≥ Y%"
- 分流：用户级随机（user_id hash），保证同用户始终同组
- 样本量：基于基线值、最小可检测效果、显著性、功效计算
- 时长：至少 1 个完整业务周期（覆盖周末/节假日效应）
- 护栏指标：核心体验指标不能下降（如退款率、投诉率）
- 判定：统计显著 + 业务显著，负面结果同样记录

### 4.3 公平性与鲁棒性

- 公平性：不同用户群（新/老/地域/性别）效果差异评估
- 鲁棒性：噪声/异常值/对抗样本下的稳定性

## 5. 模型部署与服务化

### 5.1 部署方式

| 方式 | 场景 | 工具 | 延迟要求 |
|------|------|------|---------|
| 实时推理 | 推荐/搜索/风控 | Triton / TensorFlow Serving / TorchServe / Flask/FastAPI | < 100ms p99 |
| 批量预测 | 画像/标签/报表 | Spark / Airflow + 模型批处理 | 小时级/天级 |
| 边缘推理 | 端侧推荐/图像识别 | TensorFlow Lite / ONNX Runtime / Core ML | < 50ms |

### 5.2 服务化要求

- 模型版本管理：蓝绿/金丝雀发布，支持回滚到上一版本
- 推理优化：
  - 量化（INT8/FP16）
  - 剪枝
  - 图优化（TensorRT / ONNX Runtime / OpenVINO）
  - 批处理（Batching）提升吞吐
- 超时/降级/熔断：
  - 推理超时 → 返回基线结果（热门/规则）
  - 模型异常 → 自动切换备用模型

### 5.3 Feature Store 集成

- 在线特征实时获取（Redis/Feature Store）
- 特征缺失 fallback（默认值/冷启动策略）

## 6. 监控与漂移检测

### 6.1 监控指标

- 数据漂移：输入特征分布变化（PSI / KS 检验 / Wasserstein）
- 概念漂移：特征-标签关系变化（性能衰减）
- 预测漂移：模型输出分布变化
- 系统指标：推理延迟 / QPS / 错误率 / 资源使用

### 6.2 告警与自动回滚

- 漂移阈值：PSI > 0.2 或 AUC 下降 > 5% → 告警
- 自动回滚：性能持续劣化 → 回退到上一稳定版本
- 重训练触发：
  - 定时：每天/每周
  - 事件：数据漂移超过阈值 / 业务反馈效果差

---

## 7. 迭代计划

- 反馈闭环：
  - 用户行为 → 埋点 → 数据 → 特征 → 模型 → 预测 → 用户反馈
- 迭代节奏：
  - 特征迭代：每周/双周
  - 模型迭代：每月/季度
  - 大版本：半年（架构升级/新算法）
- 实验 backlog：记录所有尝试过的特征/模型/策略，失败也记录

## 8. 验收清单（DoD）

- [ ] 业务问题、目标函数、评估指标、基线方案已明确

- [ ] 实验可追踪（代码/数据/特征/参数/指标全部版本化）
- [ ] 离线评估指标显著优于基线（统计显著 + 业务显著）
- [ ] A-B 测试方案完整（假设/分流/样本量/时长/护栏指标/判定标准）
- [ ] 推理服务有版本管理、超时降级、熔断回退策略
- [ ] 模型上线后有漂移检测、性能监控、自动告警与回滚机制
- [ ] 数据无泄露（训练/验证/测试严格分离，时间序列用前向验证）
- [ ] 公平性与鲁棒性已评估（不同用户群效果差异、噪声稳定性）
