# API 设计模板（api-design）

> 目标：让 API 一次设计正确，避免后续兼容/错误码/分页/幂等/安全问题。

## 固定输出物（AI 必须产出）

- OpenAPI/Swagger 片段（最少包含 path + request/response schema）
- 错误码表（code/含义/触发条件/是否可重试）
- 幂等策略（是否需要、幂等 key、去重窗口）
- 超时/重试/限流策略（阈值 + 行为）
- 兼容策略（新增字段默认值、弃用计划）

## 必须/禁止规则

- 必须：所有外部/下游调用设置超时
- 必须：错误响应格式稳定且可定位（code + request_id）
- 必须：明确哪些错误可重试，避免无限重试
- 禁止：返回模糊错误（如仅 "error" 或仅堆栈）
- 禁止：在 API 响应中暴露内部实现细节或敏感信息

- 超时：
  - 服务端处理超时阈值：
  - 下游调用超时阈值：
- 重试策略：
  - 哪些错误可重试？
  - 退避策略（指数退避 + jitter）
- 限流：
  - 用户级 / IP 级 / 全局
  - 触发后返回码（429）与提示
- 兼容性：
  - 字段新增：只增不删，提供默认值
  - 字段弃用：deprecated 标记 + 迁移周期
