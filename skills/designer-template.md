# 设计师能力模板（designer）

> 目标：从开发视角补全设计与交互能力，让"设计稿 → 代码"一次成型，减少还原偏差、交互遗漏、适配翻车。

---

## 固定输出物（AI 必须产出）

- 设计系统清单（色彩/字体/间距/圆角/阴影 Tokens）
- 页面/组件信息架构（层级、状态、空态、加载态、错误态）
- 交互流程图（触发条件 → 反馈 → 异常分支）
- 响应式/多端适配策略（断点、弹性规则、隐藏/重排逻辑）
- 动画与过渡规范（触发时机、时长、缓动、性能预算）
- 无障碍设计检查（ARIA、焦点管理、色彩对比度、键盘导航）
- 设计还原验收清单（像素偏差、交互一致性、动效还原度）

## 必须/禁止规则

- 必须：每个页面/组件至少定义 4 种状态（默认/悬停/激活/禁用/加载/空/错误）
- 必须：设计系统 Tokens 与代码变量一一对应（禁止硬编码颜色/字号/间距）
- 必须：交互有明确的触发条件、反馈形式、失败兜底
- 必须：关键操作有加载态和错误态（禁止点击后无反馈）
- 禁止：设计稿只出桌面端，不定义移动端适配规则
- 禁止：用 "差不多" 还原设计（关键间距/字号/颜色必须精确到 token）
- 禁止：动画无性能预算（长动画阻塞主线程、大量重排重绘）
- 禁止：忽略键盘/屏幕阅读器用户（焦点丢失、无 ARIA 标签）

## 1. 设计系统（Design System）

### 1.1 色彩系统

- 主色/辅色/功能色（成功/警告/错误/信息）
- 中性色阶（灰度 50-900）
- 暗色模式映射（light ↔ dark token 一一对应）
- 透明度/渐变规则
- 禁止：设计稿颜色与 token 不一致、开发随意改色值

### 1.2 字体与排版

- 字体栈（中文优先：系统字体 / PingFang / Noto Sans）
- 字号阶梯（xs/sm/base/lg/xl/2xl…，对应 px/rem）
- 字重规则（normal/medium/semibold/bold 使用场景）
- 行高/段间距/列表样式

### 1.3 间距与布局

- 间距 token（4px 基座：4/8/12/16/24/32/48/64）
- 栅格系统（列数/槽宽/边距/断点）
- 容器宽度（max-width、居中、全宽规则）

### 1.4 圆角与阴影

- 圆角 token（sm/md/lg/full/pill）
- 阴影层级（rest/raised/floating/modal）
- 边框规则（颜色/粗细/dash）

### 1.5 图标与插画

- 图标库（尺寸规范：12/16/20/24/32，统一 stroke-width）
- 图标状态（单色/双色/品牌色）
- 插画风格（扁平/线性/3D，统一视角和比例）

## 3. 交互设计（Interaction）

### 3.1 交互流程图

```
用户动作 → 系统反馈 → 成功分支 / 失败分支 / 异常分支
```

- 触发条件：点击/悬停/输入/滚动/手势/键盘/语音
- 反馈形式：视觉变化（颜色/大小/位置）、动效、声音、振动
- 等待策略：即时反馈 / 骨架屏 / 进度条 / 乐观更新 / 延迟加载
- 失败兜底：重试入口 / 降级展示 / 友好提示 / 日志上报

### 3.2 手势与快捷键

- 移动端：滑动方向、长按、双击、捏合、下拉刷新
- 桌面端：快捷键（Ctrl/Cmd + K 搜索、Esc 关闭、Enter 确认）
- 焦点管理：Tab 顺序、焦点陷阱（Modal/Drawer）、焦点恢复

### 3.3 微交互（Micro-interaction）

- 按钮点击：按下缩放/颜色加深
- 切换开关：滑块动画 + 状态色过渡
- 表单验证：实时反馈（对勾/感叹号 + 文字提示）
- 加载：骨架屏 / 脉冲 / 旋转 / 进度条
- 成功/失败：Toast / Snackbar / 模态确认

## 4. 响应式与多端适配

### 4.1 断点定义

| 名称 | 断点 | 布局策略 | 侧边栏 | 栅格列数 |
|------|------|---------|--------|---------|
| Mobile | < 640px | 单列堆叠 | 隐藏/抽屉 | 4 |
| Tablet | 640-1024px | 双列/折叠 | 可折叠 | 8 |
| Desktop | 1024-1440px | 多列 | 固定 | 12 |
| Wide | > 1440px | 居中+最大宽度 | 固定 | 12 |

### 4.2 适配规则

- 图片：srcset / 不同尺寸资源 / 懒加载
- 表格：横向滚动 / 卡片化 / 列优先级隐藏
- 图表：容器查询重绘 / 简化移动端视图
- 导航：汉堡菜单 / Tab / 侧边栏 / 顶部工具栏

### 4.3 安全区与异形屏

- 刘海屏/挖孔屏：安全区域 inset
- 底部手势条：底部 padding
- 横屏旋转：布局重排策略

## 5. 动画与过渡

### 5.1 动效规范

- 时长阶梯：
  - 即时反馈：50-100ms
  - 微交互：150-200ms
  - 页面过渡：300-400ms
  - 复杂动画：500-800ms
- 缓动曲线：
  - 标准：ease-in-out / cubic-bezier(0.4, 0, 0.2, 1)
  - 入场：ease-out / cubic-bezier(0, 0, 0.2, 1)
  - 退场：ease-in / cubic-bezier(0.4, 0, 1, 1)
- 性能预算：
  - 只动画 transform / opacity（避免 layout/paint）
  - 60fps 目标，长列表虚拟滚动

### 5.2 页面过渡

- 路由切换：淡入淡出 / 滑动 / 共享元素过渡
- 列表增删：FLIP 动画（位置/大小/透明度变化）
- 模态/抽屉：遮罩淡入 + 内容滑入

### 5.3 加载与骨架屏

- 骨架屏：颜色 token（比背景浅 5-10%）、脉冲动画、结构与实际内容一致
- 渐进加载：低清占位 → 高清图片、文字优先 → 图片后加载

## 6. 无障碍设计（Accessibility / a11y）

### 6.1 色彩对比度

- 正文文本：≥ 4.5:1（AA级）/ ≥ 7:1（AAA级）
- 大文本/图标：≥ 3:1
- 禁止：仅靠颜色传达状态（必须配合图标/文字）

### 6.2 键盘导航

- 所有交互元素可聚焦（tabindex）
- Tab 顺序符合视觉流向
- 焦点样式可见（outline/ring）
- 快捷键不冲突（避开浏览器默认快捷键）

### 6.3 屏幕阅读器

- 语义化 HTML（button 不是 div）
- ARIA 标签（aria-label、aria-describedby、aria-expanded、aria-live）
- 图片 alt 文本（装饰性图片 alt=""）
- 表单 label 关联（for/id 或 aria-labelledby）

### 6.4 动画与_prefers-reduced-motion_

- 尊重用户设置：减少动画（即时切换/淡入淡出）
- 无动画时保持信息完整性

## 7. 设计到代码的对接规范

### 7.1 设计稿交付标准

- 工具：Figma / Sketch / 即时设计
- 标注：
  - 所有间距、字号、颜色使用 token
  - 导出资源：1x/2x/3x、SVG（图标）、WebP（图片）
  - 组件变体：默认/悬停/禁用/加载等状态
- 交互说明：
  - 点击区域（热区 ≥ 44×44pt/px）
  - 过渡动画参数（时长/缓动/触发条件）
  - 数据状态变化（空/加载/错误/成功）

### 7.2 开发还原 checklist

- [ ] 颜色/字号/间距精确对应 token（误差 ≤ 1px）
- [ ] 所有状态（默认/悬停/禁用/加载/空/错误）已实现
- [ ] 交互反馈与 design spec 一致（时长/缓动/触发）
- [ ] 响应式在各断点无布局错乱
- [ ] 图片/图标资源清晰无拉伸
- [ ] 动画 60fps，prefers-reduced-motion 已处理
- [ ] 键盘导航顺序正确，焦点可见
- [ ] 屏幕阅读器可正确朗读关键信息

### 7.3 设计验收标准

- 像素级还原：关键页面截图对比（设计稿 vs 实现）
- 交互一致性：所有分支流程走通（正常/异常/边界）
- 性能验收：Lighthouse 性能/可访问性/SEO 评分
- 多端验收：Mobile/Tablet/Desktop 各至少 1 台真机
