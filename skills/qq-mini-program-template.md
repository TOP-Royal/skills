# QQ小程序开发模板（qq-mini-program）

> 目标：QQ小程序从开发到腾讯开放平台一次成型，覆盖社交、游戏、娱乐、工具等 QQ 生态核心场景，尤其聚焦年轻化用户群。

---

## 固定输出物（AI 必须产出）

- 产品定位与 QQ 生态契合点（社交/游戏/娱乐/工具/校园）
- 页面与信息架构（符合年轻用户操作习惯，社交属性强）
- 核心能力接入方案（登录/支付/分享/群能力/好友互动/游戏中心）
- 社交裂变与运营策略（群分享/好友助力/QQ空间/厘米秀）
- 性能与体验基线（启动/包体/渲染/内存）
- 审核与上架材料（营业执照/行业资质/隐私政策/未成年人保护）

## 必须/禁止规则

- 必须：QQ小程序必须以"社交互动"或"游戏娱乐"为核心，纯商务工具需证明有社交价值
- 必须：涉及群能力的必须有群主/管理员授权机制，禁止骚扰群成员
- 必须：游戏类必须有未成年人防沉迷（实名认证+时长限制+家长监护）
- 必须：性能基线：首屏 < 1.5s、包体 < 2MB（主包）、JS 执行 < 2s
- 禁止：诱导分享至非QQ平台（如"分享到微信"在QQ小程序内违规）
- 禁止：收集QQ好友关系链用于非社交目的（隐私红线）
- 禁止：虚拟货币/道具未接入QQ钱包支付却私下交易
- 禁止：页面空白/加载超时无反馈、无错误兜底

## 1. 平台特性与生态定位

- 核心场景：社交互动、小游戏、娱乐工具、校园服务、兴趣社群
- 用户画像：年轻化（Z世代为主）、重社交、重游戏、重娱乐
- 流量入口：
  - QQ 搜索 / 小程序中心 / 消息列表
  - 群分享 / 好友分享 / QQ空间分享
  - 游戏中心 / 厘米秀 / QQ钱包
- 与微信差异：
  - 更强社交属性（群活跃度更高，好友互动更频繁）
  - 游戏生态更成熟（QQ游戏中心导流能力强）
  - 厘米秀/QQ秀等虚拟形象联动

---

## 2. 技术架构

- 框架：QQ小程序原生框架（基于腾讯小程序框架，与微信小程序高度相似）
- 语言：JavaScript / TypeScript
- 开发工具：QQ小程序开发者工具
- 组件：qq.xxx API（与 wx.xxx 高度兼容）
- 多端复用：UniApp / Taro 支持编译到 QQ（差异极小，主要注意 API 前缀）

---

## 3. 核心能力接入

### 3.1 登录与授权

- qq.login（获取 code）→ 服务端换 session_key
- qq.getUserInfo（获取昵称/头像，需用户点击按钮）
- qq.getPhoneNumber（获取手机号，需用户主动授权）
- 群能力：
  - qq.getGroupInfo（获取群信息，需群场景打开）
  - qq.getGroupId（获取群 ID，用于群排行/群互动）

### 3.2 支付

- qq.requestPayment（QQ钱包支付）
- 接入流程：服务端创建订单 → 获取预支付信息 → 客户端调 qq.requestPayment
- 虚拟商品：必须走QQ钱包，禁止私下交易
- 退款：服务端调用QQ钱包退款接口

### 3.3 社交能力（核心差异）

- 群分享：qq.shareToQQMiniProgram（分享到QQ群/好友）
- QQ空间：qq.shareToQzone（分享到QQ空间）
- 好友互动：
  - 好友排行榜（游戏类）
  - 好友助力/邀请（运营活动）
  - 厘米秀联动（虚拟形象互动）
- 群运营：
  - 群小程序（群内常驻入口）
  - 群互动工具（投票/接龙/打卡）

### 3.4 游戏中心

- QQ游戏中心导流（小游戏首发/推荐位）
- 游戏社交：好友在玩/群排行/世界排行
- 游戏包：主包+分包，游戏资源动态加载

### 3.5 开放能力

- 扫一扫：qq.scanCode
- 位置：qq.getLocation / qq.openLocation
- 图片/文件：qq.chooseImage / qq.uploadFile
- 消息：qq.subscribeMessage（订阅消息）

---

## 4. 页面与组件规范

- 页面结构：.js / .json / .qml（模板）/ .qss（样式）
- 常用组件：view / text / image / button / input / scroll-view / swiper / canvas
- 导航：qq.navigateTo / qq.redirectTo / qq.reLaunch / qq.navigateBack
- 与微信小程序差异：
  - qml 与 wxml 语法基本一致
  - qss 与 wxss 基本一致
  - API 前缀 qq 替代 wx
  - 大部分 API 与微信兼容，少数社交特有 API（如群能力）

---

## 5. 社交裂变与运营

### 5.1 裂变机制

- 群裂变：分享到群 → 群成员参与 → 解锁奖励
- 好友助力：邀请好友助力 → 达成目标 → 获得权益
- 排行榜：群排行/好友排行 → 激发竞争/分享欲
- QQ空间：生成分享图 → 用户发空间 → 回流小程序

### 5.2 运营工具

- 消息推送：订阅消息（服务通知）/ 群消息
- 优惠券：QQ钱包卡券 / 小程序内优惠券
- 会员体系：QQ会员/超级会员权益打通

---

## 6. 审核与上架

### 6.1 资质要求

- 企业营业执照（个体工商户部分类目受限）
- 法人身份证
- ICP 备案（如涉及网络服务）
- 游戏版号（如涉及游戏）
- 未成年人保护承诺书（游戏/社交类必须）

### 6.2 审核要点

- 社交合规：禁止骚扰/诈骗/色情/赌博
- 游戏合规：版号/实名/防沉迷/虚拟货币规范
- 支付合规：走QQ钱包，禁止私下交易
- 内容合规：UGC有审核，禁止违法违规内容

### 6.3 上架流程

1. 开发完成 → 自测（开发者工具真机预览）
2. 提交审核（上传代码/填写说明/上传资质）
3. 审核中（1-5 个工作日）
4. 审核通过 → 发布上线

---

## 7. 性能与体验

- 启动优化：首屏 < 1.5s、主包 < 2MB
- 游戏性能：帧率 ≥ 30fps（小游戏）、内存及时释放
- 社交体验：分享卡片精美、群互动响应快
- 内存管理：页面隐藏时暂停动画/音频/定时器

- [ ] 所有支付走QQ钱包，无私下交易
- [ ] 群能力有授权机制，不骚扰群成员

- [ ] 社交裂变机制合规，无诱导分享至非QQ平台
- [ ] 性能达标：首屏 < 1.5s、包体 < 2MB、游戏帧率 ≥ 30fps
- [ ] 审核材料齐全（营业执照/ICP/游戏版号/未成年人保护承诺书）
- [ ] 真机测试通过（Android/iOS 多机型）
