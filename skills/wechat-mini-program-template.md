# 微信小程序开发模板（wechat-mini-program）

> 目标：把微信小程序需求一次写清、一次做对（页面/组件/权限/支付/登录/发布/性能/审核）。

---

## 固定输出物（AI 必须产出）

- 页面清单（路由/页面路径/入口）+ 关键交互流程
- 组件清单（复用组件/状态/props/事件）
- 数据接口清单（请求/响应 schema + 错误码）
- 登录与鉴权方案（code 换 openid/session、token、过期策略）
- 权限与隐私合规（用户授权弹窗、隐私协议、敏感 API）
- 性能与体验（包体、首屏、分包、图片与缓存策略）
- 发布与审核清单（版本号、体验版/正式版、审核注意项）

## 必须/禁止规则

- 必须：所有网络请求有超时 + 失败提示 + 重试策略（必要时）
- 必须：登录态、token 过期与刷新策略明确（避免线上大量 401/掉线）
- 必须：关键操作防重复提交（按钮防抖 + 后端幂等）
- 禁止：在前端存放敏感密钥（AppSecret 等只能在服务端）
- 禁止：同步阻塞或大循环卡主线程导致卡顿
